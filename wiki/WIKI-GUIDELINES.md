# Wiki Content Guidelines

This wiki folder is automatically synced to GitHub Wiki. Follow these guidelines to ensure proper formatting.

## GitHub Wiki Conventions

### File Naming
- Files must be in Markdown format (`.md`)
- File names should use PascalCase or kebab-case (e.g., `Getting-Started.md`, `API-Reference.md`)
- Special files:
  - `Home.md` - The wiki homepage
  - `_Sidebar.md` - Navigation sidebar (auto-generated by workflow)
  - `_Footer.md` - Footer content (optional)

### Linking

#### Internal Wiki Links
- Use wiki-style links: `[Link Text](Page-Name)`
- Example: `[Getting Started](Getting-Started)`
- Do NOT use `.md` extension in links
- Do NOT use full paths

#### External Links
- Use standard Markdown: `[Link Text](https://example.com)`
- Link to main README: `[Main Repository](../README.md)` or use full GitHub URL

### Images

#### Using Images in Wiki

GitHub Wiki has specific requirements for images:

**Option 1: External Hosting (Recommended)**
```markdown
![Alt Text](https://example.com/image.png)
```

**Option 2: GitHub Repository Assets**
```markdown
![Alt Text](https://raw.githubusercontent.com/YOUR_USERNAME/YOUR_REPO/main/path/to/image.png)
```

**Option 3: Wiki Repository Upload**
1. Clone the wiki repository: `git clone https://github.com/YOUR_USERNAME/YOUR_REPO.wiki.git`
2. Add images to the wiki repo
3. Reference: `![Alt Text](image.png)`

**Best Practice:**
- Use external CDN or repository raw URLs
- Keep images under 10MB
- Use descriptive alt text
- Optimize images before uploading

### Code Blocks

Use fenced code blocks with language specification:

\`\`\`php
echo "Hello World";
\`\`\`

\`\`\`bash
curl https://api.example.com
\`\`\`

### Tables

Use standard Markdown tables:

```markdown
| Column 1 | Column 2 |
|----------|----------|
| Value 1  | Value 2  |
```

### Headings

- Use `#` for main title (one per page)
- Use `##` for sections
- Use `###` for subsections
- Headings automatically create anchor links

### Best Practices

1. **Keep it simple** - Wiki is for documentation, not complex layouts
2. **No HTML** - Stick to Markdown when possible
3. **Test links** - Ensure all internal links work
4. **Update sidebar** - Keep navigation current
5. **Cross-reference** - Link related pages together

## Workflow

When you edit files in the `wiki/` folder and push to the `main` branch:

1. GitHub Actions workflow triggers
2. Files are automatically synced to GitHub Wiki
3. Sidebar is auto-generated
4. Changes appear on `https://github.com/YOUR_USERNAME/YOUR_REPO/wiki`

Manual trigger: Go to Actions → Sync Wiki → Run workflow
